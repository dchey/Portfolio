Cactus Website Analysis
Alpha/Beta Testing websites

Overview

I Alpha/Beta tested a cactus website, and statistically analyzed results to get insight into two different UI designs. I made changes to a cacti purchasing website and made two versions. Users were randomly given one of the two websites. Then I tracked the clicks and time spent on the site to get time to completion and return rate. Time to competion is the time it takes to complete the desired task, which is ordering cactis in this case. Return rate is the number of times a user returned to the main page. With the results, I analyzed if one version had more clicks or time spent with statistical analysis.

Website Differences

  • Version A: Uses more than one background and text colors.
  • Version B: Has bold text for cactus prices.

Hypothesis

Time to completion:

  • Null hypothesis: Version A and Version B will have the same time completion.
  • Alternative hypothesis: Version A and Version B will not have the same time completion.

Return rate:

  • Null hypothesis: Version A and Version B will have the same return rate.
  • Alternative hypothesis: Version A and Version B will not have the same return rate.

Data Collection

I sent the link to my peer classmates for data collection. They were randomly given one of the two versions with equal probability, and had to fill at least $150 of cacti/succulents, go to the checkout page, and come back to the main page.

Results

Time to completion:

  • t-score: 0.996312262747
  • p-value: 0.168641303251

Return rate:

  • chi^2 value: 0.405910609858
  • p-value: 0.524052450153

Since p-values are greater than 0.05 for both time for completion and return rate, we fail to reject null hypothesis.

The high p-values for both metrics tell that results are not statistically significant, which tells that the tests did not show any significant difference between A and B.

Conclusion

  • One of the limitations of this projecct was that there was not enough data points. With more data points, we could have got a better idea on how users actually behaved.
  • Another limitation was that users were not using the website; one of the datas for time of completion was a complete outlier, which tells that there is a high chance that the user is not actually doing the tasks.
  • The results show that changing or adding different colors, fonts, or styles does not significantly make the website better.
  • I learned that conclusions of more complex designs (ex. website has checkout button located top right, user can see the total price after adding/removing stuff to the cart, etc.) could be more complex because it is important to figure out how each design principle affects user experience.
